
# خريطة مشروع مائدتي (Mershhah) - نسخة مبسطة ومستقرة

## 1. وصف عام للمشروع

"مرشح" هو **الواجهة الرقمية الموحدة** للمطاعم والمقاهي. هو "الموقع الذكي" الذي يعمل داخل الفرع عبر QR Code وخارجه كرابط في البايو. المنصة مبنية على Next.js و Firebase، وتهدف إلى تبسيط التفاعل الرقمي للعملاء.

**الوظائف الأساسية:**
- **نظام مصادقة:** يعتمد على Firebase Auth لتسجيل أصحاب المطاعم.
- **لوحة تحكم للمدير:** إدارة المشتركين وتفعيل الاشتراكات يدويًا.
- **لوحة تحكم لصاحب المشروع:** إدارة المنيو، العروض، تخصيص هوية المشروع، والوصول للأدوات.
- **واجهات عملاء ديناميكية:** صفحات عامة لكل مطعم (`/hub/[username]`, `/menu/[username]`) تعرض المنيو وتوفر مساعدًا ذكيًا.
- **الذكاء الاصطناعي (AI):** استخدام Genkit و Gemini لإنشاء أوصاف للمنيو، وتقديم ردود ذكية في المحادثات.

---

## 2. شجرة المجلدات الأساسية وشرحها

```
.
├── src/
│   ├── app/                # الصفحات والمسارات (App Router)
│   ├── components/         # مكونات React
│   ├── services/           # (تم تبسيط الاعتماد عليه)
│   ├── ai/                 # منطق الذكاء الاصطناعي (Genkit Flows)
│   ├── lib/                # أدوات مساعدة، أنواع البيانات، وإعدادات Firebase
│   ├── hooks/              # React Hooks مخصصة (مثل useUser)
│   └── blog/               # محتوى المدونة
├── docs/                   # ملفات التوثيق (backend.json)
└── public/                 # (غير مستخدم حاليًا)
```

### شرح وظيفة كل مجلد:

- **`src/app`**: قلب التطبيق (App Router). يحتوي على المسارات الرئيسية: `admin`, `owner`, والصفحات العامة.
- **`src/components`**: مكونات React، مقسمة إلى:
    - `ui`: مكونات أساسية من ShadCN.
    - `shared`: مكونات مشتركة (Logo, Header, Sidebars).
    - `auth`: نماذج تسجيل الدخول والتسجيل.
    - `admin` / `dashboard`: مكونات خاصة بلوحات التحكم.
- **`src/ai`**: مركز عمليات الذكاء الاصطناعي (Genkit Flows).
- **`src/lib`**: مكتبة الأدوات المساعدة. أهمها:
    - `types.ts`: تعريفات TypeScript لأنواع البيانات.
    - `firebase.ts`: إعدادات عميل Firebase للاتصال.
- **`src/hooks`**: يحتوي على `useUser` لجلب بيانات المستخدم المسجل دخوله.
- **`docs`**: يحتوي على `backend.json` الذي يصف مخطط قاعدة البيانات.

## 3. المكونات الرئيسية والمنطق التجاري

- **نقاط الدخول:**
    - `src/app/layout.tsx`: القالب الرئيسي.
    - `src/app/page.tsx`: الصفحة الرئيسية.
    - `src/middleware.ts`: (مبسط) لا يقوم بعمليات معقدة حاليًا.

- **المنطق التجاري (Business Logic):**
    - **المصادقة (Auth):** تتم مباشرة عبر Firebase Auth في مكونات `src/components/auth`.
    - **التحقق من الصلاحيات:** يتم داخل `AccountStatusChecker.tsx` بناءً على بيانات `useUser`، ويقوم بعرض المحتوى أو حجبه بناءً على دور المستخدم وحالة حسابه.
    - **منطق الذكاء الاصطناعي:** معزول بالكامل داخل مجلد `src/ai/flows`.
    - **إدارة البيانات (Data Management):** تم إعادة هيكلة جميع الصفحات التي تستخدم `onSnapshot` لضمان عدم حدوث تسريب في الذاكرة. يتم الآن استخدام دالة تنظيف (cleanup function) بشكل صحيح وآمن في كل `useEffect`.

## 4. التقنيات المستخدمة

- **Framework:** Next.js (App Router)
- **لغة البرمجة:** TypeScript
- **قاعدة البيانات والمصادقة:** Firebase (Firestore, Auth)
- **الذكاء الاصطناعي:** Genkit, Google Gemini
- **الواجهة الأمامية (UI):** React, Shadcn/UI
- **التصميم:** Tailwind CSS

---

## 5. خريطة تدفق البيانات (مثال: عرض تفاصيل مشترك)

1.  **المستخدم (المدير)**: يزور صفحة `/admin/management`.
2.  **React (Client Component):**
    -  `ManagementPage` يقوم بجلب قائمة المشتركين.
    - عند اختيار مشترك، يتم تمرير `profileId` إلى المكون الداخلي `ProfileDetails`.
3.  **`ProfileDetails` Component:**
    - يستخدم `useEffect` مع `profileId` كـ dependency.
    - داخل `useEffect`، يتم إنشاء `onSnapshot` listener لجلب تفاصيل المشترك المختار وبياناته المالية.
    - **الأهم:** يتم إرجاع دالة تنظيف `unsubscribe` من `useEffect`، والتي يتم استدعاؤها تلقائيًا عند تغيير `profileId` أو مغادرة الصفحة، مما يمنع تسريب الذاكرة والانهيارات.
4.  **Firestore Security Rules:** تتحقق من أن المستخدم مدير قبل السماح بقراءة بيانات المشتركين.
5.  **UI Update:** يتم تحديث واجهة المستخدم لعرض تفاصيل المشترك المختار بأمان.
