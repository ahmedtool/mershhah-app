# حالة المشروع الحالية - منصة مرشح (Mershhah) - بعد إعادة الهيكلة

## 1. نظرة عامة
مرشح هو **الواجهة الرقمية الموحدة** للمطاعم والمقاهي، تعمل داخل الفرع عبر QR Code وخارجه كرابط ذكي. المنصة مبنية على ركيزتين:
1.  **واجهة عميل ذكية:** تجمع كافة روابط المشروع (منيو، عروض، فروع، إلخ) في صفحة واحدة.
2.  **منصة أدوات داخلية:** "صندوق أدوات" لصاحب المشروع يحتوي على أدوات تحليلية وإبداعية.

## 2. الهيكل التقني (Tech Stack)
- **Framework:** Next.js 15 (App Router)
- **Styling:** Tailwind CSS + Shadcn/UI
- **Database & Auth:** Firebase (Firestore, Authentication)
- **AI:** Google Genkit + Gemini
- **Language:** Arabic (RTL support)

## 3. تقسيم النظام (Roles & Dashboards)

### أ- لوحة تحكم المدير العام (Admin Dashboard)
- **إدارة المشتركين:** تفعيل الحسابات والاشتراكات يدويًا، إدارة الصلاحيات.
- **إدارة المحتوى:** إدارة الباقات، الأدوات، التطبيقات، والإعلانات.

### ب- لوحة تحكم صاحب المشروع (Owner Dashboard)
- **الإحصائيات:** نظرة سريعة على تفاعل العملاء.
- **إدارة المحتوى:** التحكم في المنيو، العروض، والفروع.
- **متجر الأدوات:** تفعيل أدوات إضافية.
- **الاستوديو:** تخصيص الهوية البصرية وإنشاء صور إعلانية.

## 4. حالة التطوير الحالية
- [x] **إعادة هيكلة شاملة:** تم إعادة كتابة الأجزاء الرئيسية من الكود لضمان الاستقرار.
- [x] **حل مشاكل Firebase:** تم حل مشاكل `unsubscribe` و `memory leak` بشكل جذري في جميع الصفحات التي تستخدم `onSnapshot`.
- [x] **تبسيط الكود:** تم حذف الملفات والمكونات غير الضرورية، وتم تبسيط منطق جلب البيانات.
- [x] **استقرار النظام:** المشروع الآن في حالة مستقرة وخالية من الأخطاء المتكررة التي كانت تسبب انهيار الصفحات.
- [x] لوحة تحكم المدير تعمل بشكل صحيح، مع تفعيل يدوي للاشتراكات.
- [x] واجهات صاحب المشروع تعمل وتعتمد على بيانات حية بشكل آمن.

## 5. نموذج العمل الحالي
- **الاشتراك:** ثلاث باقات ثابتة (مجانية، ربع سنوية، سنوية).
- **التفعيل:** يتم تفعيل الاشتراكات يدويًا من قبل المدير بعد التحقق من الدفع عبر رابط خارجي.
- **الإضافات:** متجر أدوات يمكن تفعيله.
